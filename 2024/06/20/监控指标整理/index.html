监控指标整理

  1. 主机级别：

  • node.alive：主机存活，1表示存活，0表示不存活；

  • ssh.status：ssh状态，1表示ssh状态正常，0表示不正常；

  1. 主机CPU：

我们从top指令中可以拿到类似如下的结果：

这里的每个值（9.8，5.8等）都是占用cpu的百分比，具体的含义是：

9.7 us: 用户空间（user space）占用的 CPU 百分比。用户空间是指运行用户进程的时间。 5.8 sy: 内核空间（system space）占用的 CPU 百分比。内核空间是指运行内核进程和系统调用的时间。 0.0 ni: 改变过优先级的用户进程占用的 CPU 百分比。ni 代表“nice”值。 80.6 id: CPU 空闲时间的百分比。80.6% 的时间 CPU 是空闲的。 0.0 wa: 等待 I/O 操作完成的时间占用的 CPU 百分比。wa 代表“wait”。 3.8 hi: 硬件中断占用的 CPU 百分比。hi 代表“hardware interrupts”。 0.1 si: 软件中断占用的 CPU 百分比。si 代表“software interrupts”。 0.0 st: 被虚拟化环境中的其他任务偷走的 CPU 时间百分比。st 代表“steal”。

这些值加起来，理论上是100%，也会因为精度问题略小于100%。
